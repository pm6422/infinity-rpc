LUI️X RPC - Remote Procedure Call Framework

A remote call framework in a distributed environment that makes remote calls as simple as local calls

LUI️X RPC - Service Governance System

Provides a suite of powerful service governance tools, including application management, server monitoring, service provider and consumer management, and monitoring features

Interface-Oriented RPC Calls

TCP-based, interface-oriented remote procedure calls that simplify remote call logic for clients. Clients can make complex server calls with a simple local method call, and the framework is minimally invasive to the code. RPC uses a persistent connection, reducing network overhead, making it highly performant and suitable for intranet transmissions.

Service Registration and Auto-Discovery

Applications automatically register their IP address to the registry at startup, avoiding tedious network configuration. The registry then informs other services of these addresses, enabling automatic discovery among applications, improving query efficiency as the number of services grows.

Fault Tolerance and Load Balancing

The client retrieves a list of server IP addresses from the registry and selects an appropriate address for remote calls using load balancing algorithms, such as random, sticky, and low-load call. In case of failure, the next steps are based on the configured fault tolerance strategy, which includes fail-fast, failover, etc.

High-Performance Protocol and Serialization

Utilizes a custom high-performance TCP-based protocol suitable for small message data exchanges, fulfilling most interface needs. The framework includes multiple popular serialization and deserialization algorithms, like Kryo and Hessian, and can intelligently select the optimal method for each data type.

Intelligent Parameter Configuration

Designed for powerful functionality and ease of use, the system minimizes external parameter configurations to avoid selection difficulties and complexity. Instead, algorithms automatically select the optimal parameters, freeing the user's mind from configuration overload.

Service Governance Control Center

Offers a powerful web-based service governance control center - LUI️X RPC Web Center, which allows for service management and monitoring, dynamic configuration updates, interface publishing and unpublishing, and more. These operations are transparent to the client, providing a unified experience.

Additional Features

More and more extra great features

Multi-Type & Multi-Quantity Registry Centers

Supports Consul and ZooKeeper registries and reserves extension interfaces for future integrations. An application can also register with multiple registries of different types simultaneously.

Multi-Form Interfaces

A single service interface can have multiple implementation classes to meet different needs, all coexisting permanently. Example: separate implementations for mobile and PC versions.

Retry Count and Request Timeout

Sets retry count for repeated requests in case of RPC call failures, along with a request timeout. If a response isn’t received within the timeout period, the call is treated as a failure.

Direct Call

The client can discover and call services through the registry, or skip the registry and call the server directly, similar to an HTTP call using the configured server address.

Intelligent Serialization & Deserialization

The system automatically chooses the most suitable serialization algorithm based on data type, bypassing incompatible algorithms like Hessian if Serializable is not implemented.

Scheduled Tasks

The LUIX RPC Web Center includes a scheduling system that uses CRON expressions or fixed intervals to initiate RPC calls, with support for timeout settings, retries, fault tolerance, and reports.

Concurrency Control & Rate Limiting

The system offers traffic control capabilities, including algorithms to limit client and server concurrent threads, along with multiple request limit algorithms, such as QPS rate limiting.

Multi-Version Interfaces

A single service interface can have different implementation versions, a temporary coexistence measure for seamless user experience during production upgrades.

Broadcast Calls

Clients can issue a single request that simultaneously calls all server programs under an interface, ideal for scenarios like synchronizing cache values across nodes.

Generic Calls

Clients can complete RPC calls without server interface dependencies, useful for cases like PHP clients calling Java interfaces without importing Java dependencies.

Visual Monitoring & Reporting

The LUIX RPC Web Center lists all applications, services, and server hardware configurations and offers graphs for service dependency, scheduled tasks, and various reports.

Automatic & Manual Service Downgrade

The LUI️X RPC Web Center provides multiple automatic service downgrade configurations, including scheduled and conditional downgrades, and also supports manual downgrade.

Product Design Concepts

Product ideas

LUI️X RPC

Simplicity

The product design is guided by simplicity from beginning to end. Designed so that even fresh graduates can quickly get started, LUI️X RPC Web Center requires no database configuration. Just run a single command to auto-download, install, and start the application.

LUI️X RPC

Intelligence

With numerous parameters and values, manual selection is often confusing. LUI️X RPC uses algorithms to automatically choose the best serialization algorithm, freeing users from such complex decisions.
