面向接口的RPC调用

基于TCP协议、面向接口的远程过程调用，系统通过屏蔽复杂的远程调用逻辑，客户端通过最简单的本地方法调用方式就可以完成复杂的远程服务端程序的调用。RPC使用长链接，不必每次通信都要三次握手，减少了网络开销，适用于内网传输。

服务注册与自动发现

应用启动后无需做相关网络配置，应用会自动将自身IP地址注册到注册中心。注册中心会通知到其他所有服务，因此注册中心下的每一个应用可以自动感知发现其他所有的应用服务。当服务数量到一定级别时，通过注册中心查询服务的效率会更高。

容错与负载均衡

客户端通过注册中心拿到一组服务地址，因此需要负载均衡算法来选择合适的地址发起调用，负载均衡算法包含：随机、粘性、低负载方调用等。 调用失败则需要根据容错策略来判断下一步处理，服务调用容错策略包含：failfast、failover等。

高性能协议与序列化算法

使用基于高性能的TCP长链接而自研的协议，适用于小报文数据交换场景，满足于我们日常工作中的绝大多数接口。并且系统内置多种市面上常见的高性能序列化和反序列化算法，程序可以根据不同的数据类型智能地选择出最优的序列化和反序列化算法。

智能化参数配置

本系统设计目标是功能强大、使用简单。系统不对外提供过多的参数配置，如果参数过多的话会带来选择困难问题，如果对外提供数百个参数值，没有深入了解该系统源码的应该也不会选。因此程序根据算法自动选择出最优的参数值，解放使用者的大脑。

服务治理控制中心

提供一个强大功能的基于Web的服务治理控制中心系统 - LUI️✘ Web Center，可以通过该控制中心管理和监控所有服务，同时也可以动态修改服务的相关配置参数。通过服务治理控制中心可以实现丰富的监控管理，还可以发布、下线接口、动态扩展等，对调用方来说是无感知、统一化的操作。

其他特性

More and more extra great features

多类型与多数量注册中心

支持ZooKeeper和Nacos注册中心，并且预留扩展接口，未来可以接入更多类型的注册中心。此外同一个应用可以注册到多个注册中心也可以注册到多种不同类型的注册中心。

多形式接口

同一个服务接口针对不同需求可以有多个实现类，这些实现所提供的服务是永久并存的，它们可以同时注册到注册中心。使用场景：如同一接口有手机端和PC端不同实现。

重试次数与请求超时

重试次数用于调用重试，当RPC调用失败后会重复发送请求直到成功或到达重试次数时为止。另外可以设置请求超时时间，超过该时间还未获得响应则按照调用失败处理。

直连调用

客户端应用可以通过注册中心发现服务端服务完成RPC调用，同时客户端也可以不使用注册中心，使用类似于HTTP调用方式直接通过已配置的服务端地址调用服务端程序。

智能化的序列化与反序列化

系统可以根据请求参数或返回结果的数据类型自动选取合适的序列化算法，如请求参数类型没有实现Serializable接口，系统就不会选择Hessian，而选择一个合适的序列化算法。

定时任务

LUI️✘ Web Center内置了一个定时调度系统，系统可以根据CRON表达式、固定间隔模式定时发起RPC调用。另外支持超时时间设置、重试次数、容错以及报表统计。

请求并发控制与数量限制

系统有一定的流量控制能力，提供多种请求并发控制算法，包含客户端和服务端的最大并发执行线程数量控制。并提供多种请求次数限制算法，包含QPS限流算法等。

多版本接口

同一个服务接口可以有多个不同版本的实现类，使用类似于多形式接口。但其目的是为了生产环境升级不影响用户体验，将原有老版本实现替换掉，是一种临时共存措施。

广播调用

支持客户端执行一次请求同时调用同一个接口对应的所有服务端程序进而实现广播的效果，使用场景：如将应用本地缓存值更新，如果不使用广播会导致某些节点缓存数据不一致。

通用调用

客户端支持在不引入服务端的接口依赖包而通过通用调用完成RPC调用，使用场景：如PHP工程师想要调用Java接口，但是PHP程序中无法引入Java服务端依赖包。

可视化监控及报表统计

LUI️✘ Web Center可以列出所有应用、服务、服务器乃至服务器硬件配置，同时通过可视化图形界面提供服务间依赖关系图、定时任务每日执行次数统计等一系列的各种图表。

自动与手动服务降级

LUI️✘ Web Center提供了多种自动服务降级规则配置，包含：定时降级、条件降级等配置，满足规则时系统会自动进行服务降级。同时也提供手动服务降级的功能。

产品设计理念

Product ideas

LUI️✘

简单

简单的理念至始至终贯穿于整个产品设计。设计初衷就是让刚毕业的大学生都可以很快的上手。未来会提供详尽的开发者文档设计、丰富的代码示例。 LUI️✘ Web Center甚至不需要做任何的数据库配置，只需执行一行命令，系统就可以自动下载数据库并安装配置好后自动启动应用。

LUI️✘

智能

系统不对外提供大量的参数配置，如果对外开放几十个参数，每个参数又有几个参数值。对于数百个参数值的选择也是头疼问题。例如：系统内置kryo, hessian, protobuf等多种算法， hessian支持某种类型，但是又不支持另外一种类型，那使用者到底如何选择？所以根据我们智能化理念，全部是由程序自动判定选择最合适的序列化算法。
